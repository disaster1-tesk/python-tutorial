# 提示工程 (Prompt Engineering)

## 1. 提示工程基础

### 知识点解析

**概念定义**：提示工程是指设计和优化输入给大语言模型（LLM）的提示词（Prompt），以获得更好的输出结果的技术。它是人与 AI 沟通的核心技能。

**核心概念**：
- **Prompt**：给 LLM 的输入文本，包含指令和上下文
- **Few-shot Learning**：通过示例帮助模型理解任务
- **Zero-shot Learning**：不提供示例，仅靠指令完成任务
- **Chain-of-Thought (CoT)**：引导模型逐步推理

**核心规则**：
1. 清晰、具体的指令比模糊的指令效果更好
2. 使用分隔符清晰区分指令和上下文
3. 给模型分配角色可以改善输出
4. 负向约束（告诉模型不要做什么）同样重要

**常见易错点**：
1. 指令过于笼统，模型无法准确理解意图
2. 上下文信息不足或不清晰
3. 没有考虑模型的输出格式限制
4. 忽略模型的推理能力，未充分利用 CoT

### 实战案例

#### 案例1：基础提示词
```python
# 不好的提示词（过于简单）
bad_prompt = "写一首诗"

# 好的提示词（清晰具体）
good_prompt = """请写一首关于春天的七言绝句，要求：
1. 描绘春天的景色
2. 押韵工整
3. 意境优美
"""

# 调用 LLM
from langchain_openai import ChatOpenAI
llm = ChatOpenAI(model="gpt-3.5-turbo")
response = llm.invoke(good_prompt)
print(response.content)
```

#### 案例2：角色扮演提示词
```python
role_prompt = """你是一位资深的Python工程师，有10年的开发经验。
你的风格是：
1. 代码简洁规范
2. 注重性能优化
3. 善用类型注解
4. 注释详尽

请用Python实现一个快速排序算法，要求：
- 时间复杂度最优
- 包含完整的类型注解和文档字符串
- 添加性能测试代码
"""

response = llm.invoke(role_prompt)
print(response.content)
```

## 2. Few-shot 提示技巧

### 知识点解析

**概念定义**：Few-shot Learning 通过提供少量示例来帮助模型理解任务要求和输出格式。

**核心概念**：
- **示例（Example）**：展示输入和期望的输出对
- **任务描述**：说明要完成什么任务
- **格式规范**：定义输出的结构

**核心规则**：
1. 示例应具有代表性，覆盖常见情况
2. 示例数量通常 3-5 个为宜
3. 示例与实际任务应风格一致
4. 可以用占位符替代重复内容

**常见易错点**：
1. 示例过多导致上下文过长
2. 示例格式与期望输出不一致
3. 示例不够典型，无法覆盖主要场景

### 实战案例

#### 案例1：情感分析 Few-shot
```python
few_shot_prompt = """对以下文本进行情感分析（positive/negative/neutral），只输出情感标签。

示例：
文本: 这个产品太棒了，非常满意！
情感: positive

文本: 体验很差，不推荐购买
情感: negative

文本: 还可以，一般般吧
情感: neutral

请分析：
文本: 刚刚入手了这款手机，拍照效果超级棒，满意！
情感:"""
```

#### 案例2：数据提取 Few-shot
```python
extraction_prompt = """从以下文本中提取关键信息，输出JSON格式。

示例：
文本: 张三，男，1985年3月15日生于北京，现任阿里巴巴高级工程师
输出: {"name": "张三", "gender": "男", "birth": "1985-03-15", "birthplace": "北京", "company": "阿里巴巴", "title": "高级工程师"}

文本: 李四女士是华为公司的项目经理
输出:"""
```

## 3. Chain-of-Thought (CoT) 提示

### 知识点解析

**概念定义**：思维链提示是一种引导模型进行逐步推理的技术，通过显式展示推理过程来提高模型在复杂任务上的表现。

**核心概念**：
- **推理步骤**：将复杂问题分解为多个简单步骤
- **中间结果**：展示每一步的推理结果
- **自洽性**：模型在推理过程中保持逻辑一致

**核心规则**：
1. 在提示中加入"请一步步思考"或"展示你的推理过程"
2. 提供带有推理过程的示例
3. 对于数学问题，效果尤为显著
4. 可以结合 Few-shot 使用

**常见易错点**：
1. 简单问题不需要 CoT，反而增加复杂度
2. 模型可能跳过步骤直接给答案
3. 推理步骤需要逻辑连贯

### 实战案例

#### 案例1：数学推理
```python
cot_prompt = """请一步步计算以下问题，并展示推理过程。

问题：小明有50个苹果，小红比小明多15个，小华比小红少10个。小华有多少个苹果？

推理过程："""

# 也可以用 Few-shot + CoT
cot_fewshot_prompt = """请一步步计算并给出答案。

示例：
问题：小明有10个球，小红比他多5个，小华比小红少3个。小华有几个球？
推理：小明有10个球，小红有10+5=15个，小华有15-3=12个。
答案：12个

问题：小张有100元，买了书花了35元，又赚了20元，现在有多少钱？
推理："""
```

#### 案例2：逻辑推理
```python
logic_cot_prompt = """请分析以下逻辑问题，展示你的推理过程。

问题：有A、B、C三个人。A说B是骗子。B说C是骗子。C说A和B都是骗子。已知只有一个人说的是真话，请问谁说的是真话？

推理过程：
1. 假设A说的是真话，那么B是骗子，B说C是骗子就是假的，所以C不是骗子。
2. 但C说A和B都是骗子，如果A说的是真话，那么B是骗子，A不是骗子，这与C的话矛盾。
3. ...（继续推理）

结论："""
```

## 4. 结构化输出提示

### 知识点解析

**核心概念**：
- **JSON Schema**：定义输出的数据结构
- **Markdown 格式**：使用 Markdown 组织输出
- **分隔符**：使用特殊标记区分不同部分

**核心规则**：
1. 明确指定输出格式
2. 提供格式示例
3. 使用约束确保格式正确
4. 可以结合 LangChain 的 OutputParser

### 实战案例

#### 案例1：JSON 输出
```python
json_prompt = """请将以下信息提取为JSON格式。

要求：
- 使用标准的JSON格式
- 字段名使用英文
- 日期格式为YYYY-MM-DD

文本：王五，1990年5月1日加入腾讯，担任高级开发工程师

输出："""

# 更严格的约束
strict_json_prompt = """请按以下JSON Schema输出：
{"type": "object", "properties": {"name": {"type": "string"}, "age": {"type": "number"}}, "required": ["name", "age"]}

文本：小明今年25岁
输出JSON：
"""
```

#### 案例2：Markdown 格式输出
```python
md_prompt = """请用Markdown格式总结以下内容：

# 格式要求
## 主要特点
- 列出3个要点

## 应用场景
- 列出2个场景

## 示例代码
```python
# 代码块
```

内容：Python是一种高级编程语言
"""

# 使用表格
table_prompt = """请用表格对比以下内容：

| 语言 | 特点 | 适用场景 |
|------|------|----------|

| C++ | 高性能 | 系统编程 |

请补充 Rust 的信息：
"""
```

## 5. 高级提示技巧

### 知识点解析

**核心概念**：
- **Self-Consistency**：自洽性，多次采样取最一致答案
- **Tree of Thoughts**：思维树，探索多种推理路径
- **ReAct**：推理+行动，结合工具使用
- **Prompt Chaining**：提示链，多步骤任务分解

### 实战案例

#### 案例1：分步提示链
```python
# 步骤1：提取关键信息
step1_prompt = """从以下文本中提取关键实体（人物、地点、组织）：

文本：马云于1999年在杭州创立了阿里巴巴，现在阿里巴巴是全球最大的电商公司之一。

实体："""

# 步骤2：基于提取的信息生成内容
step2_prompt = """基于以下实体，生成一段介绍：

实体：["马云", "杭州", "阿里巴巴"]

要求：生成一段关于这些实体关系的介绍（150字）：
"""

# 步骤3：验证和优化
step3_prompt = """请检查以下内容是否准确：

内容：马云于1999年在杭州创立了阿里巴巴。

如有错误，请指出并更正：
"""
```

#### 案例2：约束提示
```python
constraint_prompt = """请生成一篇技术文章，要求：
1. 字数控制在500-800字
2. 使用中文
3. 包含至少3个小标题
4. 不能出现以下词汇：绝对、完美、毫无
5. 结尾要有总结

主题：Python的优势
"""

# 负面约束示例
negative_prompt = """请写一段代码，要求：
- 不使用for循环
- 不使用递归
- 使用列表推导式
- 函数要有类型注解

实现一个求和函数：
"""
```

## 6. 提示工程最佳实践

### 常见模式

1. **角色+任务+格式+约束**
   ```
   你是[角色]，请[任务]，输出格式为[格式]，注意[约束]
   ```

2. **示例+任务**
   ```
   [示例1]
   [示例2]
   [示例3]
   请完成：[任务]
   ```

3. **思考过程+结论**
   ```
   问题：[问题]
   请一步步思考，然后给出结论。
   ```

### 调试技巧

1. **分层调试**：先验证任务理解，再优化输出格式
2. **变量替换**：用具体值替换占位符测试
3. **对比测试**：用不同提示词对比输出
4. **迭代优化**：基于输出持续改进提示词

### 性能优化

1. 简洁有效的提示比冗长更好
2. 关键信息放在开头
3. 使用具体而非抽象的描述
4. 考虑模型的 token 限制
